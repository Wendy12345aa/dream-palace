# UI and Art Memory

## Interaction Language

- Scene first, character first, context first.
- Morning Court is gameplay and the home screen.
- Kingdom Table is a physical shared workbench, not a traditional map page.
- Important operations should be in-world actions: ledger, blueprint, model, flag, seal, evidence, announcement.
- HUD is minimal and contextual. Persistent resource panels are discouraged.
- One frame should usually focus on one or two companions.

## Visual Identity

- Premium anime-inspired Eastern fantasy with an original identity.
- Warm, hopeful, elegant, cinematic, lived-in.
- Characters before environment; environment before UI chrome.
- Clear silhouettes, restrained ornament, natural materials, air and sunlight.
- Living details: moving cloth, smoke, notes, construction, changing light, accumulated history.
- Use inspiration for philosophy and quality, never direct visual imitation.

## Dream Palace Screenshot Test

A strong frame satisfies at least three:

- companion visible
- warm light
- visible history or consequence
- living background
- emotional focus
- camera storytelling
- minimal UI
- environmental motion

## Never

- Dashboard, card grid, floating panel stack, permanent sidebars.
- Page-game or generic mobile-strategy composition.
- One-note gold/brown palette, grimdark scene, excessive particles.
- Character lineup used as ordinary gameplay.
- UI that covers faces or competes with the focal interaction.
- Chibi or generic AI-anime drift for production companion art.

## Responsive Rules

- Protect character faces, dialogue, and the current physical interaction first.
- Hide nonessential companion staging while the mobile workbench/result needs space.
- Preserve stable dimensions so labels and state changes do not shift layout.
- Test desktop and portrait mobile for overflow and incoherent overlap.
