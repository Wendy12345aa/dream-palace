# Dream Palace Asset Types

Use this file to decide requirements by asset type.

## Opening CG

Use for story moments in the demo opening.

Default:

- 16:9.
- No UI, text, logo, or watermarks.
- Cinematic composition.
- Story cue visible within the image.
- Warm, urgent-but-hopeful mood.

Known opening CG sequence:

1. `cg_opening_01_city_before_dawn.webp`: living capital before sunrise; small messenger carrying petition toward palace.
2. `cg_opening_02_petition_arrives.webp`: sealed, worn petition moving through palace gate/corridor.
3. `cg_opening_03_mp_ledger.webp`: MP working over ledger and emergency request.
4. `cg_opening_04_tab_review.webp`: Tab reviewing/stamping risk document.
5. `cg_opening_05_kel_blueprint.webp`: Kel unfolding bridge or infrastructure blueprint.
6. `cg_opening_06_court_turns_to_player.webp`: court turns toward player, face unseen, Morning Court begins.

## Background

Use for static/dynamic scene layers.

Default:

- 16:9 unless used as parallax layer.
- No text.
- Leave visual room for characters and dialogue UI.
- Environment should feel lived-in.

Key demo backgrounds:

- Morning Court.
- Palace corridor or garden.
- Treasury or ledger table.
- Workshop corner.
- Village or kingdom exterior.
- Evening palace view.

## Character Portrait

Use for dialogue and companion interaction.

Default:

- Transparent background.
- Half-body preferred for dialogue; full-body only when requested.
- 3/4 view or front-facing depending on use.
- No text, no frame, no opaque background.
- Preserve character identity from `character-visuals.md`.

## Anonymous Player / Ruler

Use only when the player must be visually implied.

Default:

- Gender-ambiguous or face-hidden.
- Do not fix face, age, or identity.
- Do not sit as a fully defined emperor/empress.
- Use hands, sleeves, silhouette, veil, back view, face shadow, or foreground point-of-view composition.
- Should not overpower companions.

## UI Asset

Use for frames, dialogue panels, choice buttons, notifications, ledgers, seals, documents.

Default:

- Transparent background unless the asset is a full-screen mockup.
- No baked-in text unless it is purely decorative and approved.
- Leave safe content area.
- Make the UI feel like a court object, not a web card.

Examples:

- `dialogue_frame_gold.webp`
- `choice_button_primary.webp`
- `ui_ledger_panel_mp.webp`
- `ui_stamp_approved_tab.webp`
- `ui_blueprint_frame_kel.webp`

## Icon

Default:

- Transparent background.
- Readable at small sizes.
- Use Dream Palace material language: ink, seal, ledger, lantern, coin, grain, blueprint, ribbon.
- Avoid generic game-icons style if it breaks project identity.

## Effect

Default:

- Transparent background.
- Loopable if intended for animation.
- Subtle and emotional, not spectacle.

Common effects:

- sunlight
- dust
- petals
- wind
- rain
- candlelight
- snow
- ink
- paper
- lantern glow

## Key Visual

Use for GitHub hero, Steam, website, pitch materials.

Default:

- Composition should communicate empty throne/player place, Morning Court, companions, warm light, and a kingdom worth returning to.
- Do not make a crowded lineup unless the user explicitly asks.
- Prefer cinematic staging and character relationships.
