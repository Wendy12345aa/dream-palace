# Dream Palace Assets

This folder is reserved for shared project assets used by the README, Steam page, website, demos, and documentation.

## Planned Hero Asset

- `key-visual.png`: official Dream Palace key visual for GitHub, Steam, and the website.
- `logo.png`: official Dream Palace logo.

Until final art exists, avoid committing temporary AI mockups as canonical brand assets.

## Reference Concepts

Non-canonical visual references live under `reference/concepts/`.

- Use lowercase kebab-case English filenames.
- Organize character work as `characters/<character-id>/` and scenes as `scenes/`.
- Include the subject, asset type, `concept`, and a version suffix such as `v01`.
- Keep uncertain character assignments under `characters/unassigned/` until the
  Character Bible and art direction confirm their identity.
- Remove byte-identical duplicates rather than adding a second filename.

See [`reference/concepts/README.md`](reference/concepts/README.md) for the current
inventory and review notes. Concept references must not replace `key-visual.png`
or `logo.png` without explicit art-direction approval.
